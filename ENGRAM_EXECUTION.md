# ENGRAM Execution
## Chronological operating order for the current phase

## Phase 0: Foundation

1. Lock the narrative
2. Produce one elite sample audit
3. Build a qualified lead list
4. Set up the operator assets in `assets/`

### What "locked" means

- One ICP
- One pricing ladder
- One disclaimer
- One metric system
- One client deliverable description

## Phase 1: Validation sprint

Run this as a disciplined 14-day sprint.

Use the two core agents throughout this phase:

- Outbound Commander for pipeline, research, personalization, and outbound
- Proof Architect for the sample audit, report polish, and proof assets

### Days 1 to 3

- Audit one real beauty or skincare ad
- Turn it into a polished sample PDF
- Build a 25-lead list of qualified agencies

### Days 4 to 7

- Send 15 personalized outreach messages
- Post one sample insight publicly
- Track replies and patterns

### Days 8 to 10

- Follow up non-replies
- Take discovery calls
- Show the sample audit live instead of over-explaining the business

### Days 11 to 14

- Close one paid pilot
- Deliver within 48 hours

### Success gate

- 20 or more qualified outreaches
- 5 or more replies
- 3 or more calls
- 1 paid pilot

If the sprint misses the gate, revise the message and proof asset before adding more tooling.

## Phase 2: Early revenue

- Deliver 3 to 5 audits manually
- Log every engagement
- Identify which finding triggers the strongest client reaction
- Use that insight in future outreach and positioning

## Phase 3: Retainer conversion

After repeated use with one agency:

- pitch ongoing launch support
- include monthly category trend context
- keep turnaround fast

## Phase 4: Automation

Only automate proven bottlenecks after revenue validates demand.

Strict order:

1. intake automation
2. transcript extraction
3. scene segmentation
4. response scoring pipeline
5. semantic mapping pipeline
6. recut brief drafting support
7. report population support

## Phase 5: Agentic loops

Once the manual service is stable, turn the highest-value recurring work into loops with clear outputs.

Priority loops:

1. outreach intelligence
2. skill refinement
3. market signal capture
4. proof asset generation
5. business oversight review
6. design language and client-experience review

Core agent pairing:

- Outbound Commander owns momentum toward conversations and closes
- Proof Architect owns trust, polish, and proof quality

Each loop should:

- read a real source file
- write a real output file
- leave a reviewable next action
- stay bounded enough to rerun weekly without chaos

## Operating rule

Do not build software because it sounds impressive. Build only what reduces audit time, improves consistency, or increases margin after the manual service is already proving demand.
