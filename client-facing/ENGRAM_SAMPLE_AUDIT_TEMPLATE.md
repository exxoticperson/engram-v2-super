# ENGRAM Sample Audit Template
## Temporary live baseline until the new diagnostic report is provided

This file is the current operator template for the report structure. It is live enough to sell and deliver now, but the exact page language should be updated after the final diagnostic report is supplied.

## Page 1: Cover

- Brand
- Platform
- Category
- Date
- Confidence label
- Subtitle: "Where this creative loses the viewer, why, and what to fix before media spend."

## Page 2: Executive Summary

- Verdict: GO / CONDITIONAL GO / NO GO
- ENGRAM Score out of 100
- Confidence label
- Strongest elements
- Weakest elements
- Biggest issue
- Fastest win
- One-sentence summary

## Page 3: Category Trend Snapshot

- What is winning in the category right now
- What is getting oversaturated
- Cultural-Neural Alignment summary
- Data source label: Real-time or Benchmark-Based

## Page 4: Response Curve

Report the five zones:

1. Hook Window
2. Attention Retention
3. Product or Value Reveal
4. Engagement Sustain
5. CTA Window

For each zone:

- rating
- timestamp range
- what is happening

Add curve summary:

- first major drop
- best moment
- weakest moment
- overall hold quality

## Page 5: Diagnostic Breakdown

Include all nine metrics:

1. Hook Friction
2. Comprehension Drift
3. Reveal Waste
4. CTA Recall Gap
5. Modality Conflict
6. Branding Delay
7. Edit Saturation
8. Dead Air Window
9. Cultural-Neural Alignment

For each:

- status
- timestamp if applicable
- severity
- one-sentence diagnosis

## Page 6: ENGRAM Scorecard

Score these nine dimensions:

- Attention Continuity
- Semantic Clarity
- Memory Salience
- Reveal Timing
- CTA Retention
- Modality Alignment
- Brand Imprint
- Edit Efficiency
- Cultural-Neural Alignment

Include the composite score and interpretation band.

## Page 7: Repair Recommendations

Limit to the top three high-severity issues.

For each:

- issue
- diagnosis
- exact fix
- expected impact

## Page 8: Recut Brief

- recommended edit sequence
- three hook variants
- two CTA variants

Keep this as a written direction set for the editor or creative team. Do not imply that ENGRAM is generating replacement footage in V1.

## Page 9: Launch Recommendation

- final verdict
- required conditions before launch
- best use case for the current or edited cut
- proof policy statement

## Confidence labels

- HIGH: final cut plus performance data
- MEDIUM: final cut without performance data
- LOW: rough cut, storyboard, script, or pre-production
- CONTEXT-LIMITED: description only

## Required proof policy

> ENGRAM diagnostics are based on neural response prediction and semantic analysis signals. They identify likely attention failures and comprehension gaps. They are not guarantees of business outcomes. This is a decision-support tool designed to improve launch selection and reduce creative waste, not a predictor of conversion rate, ROAS, or sales.
