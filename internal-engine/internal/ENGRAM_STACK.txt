# ENGRAM Internal Stack
## Internal-only notes for the current and future build

This file is for internal operators and technical setup planning only.

## Current operating principle

The live business is manual. Internal tools exist to improve quality and speed, not to redefine the public product.

## Current tool categories

- structured creative analysis by the analyst
- semantic interpretation support
- optional response-model support
- trend research support
- rewrite and repair drafting support

## Public/private split

Public:

- problem
- diagnosis
- repair
- speed
- decision support

Internal:

- model names
- APIs
- research papers
- pipeline design
- thresholds

## Stack posture by phase

### Phase 1

- manual audits
- optional technical assist
- no automation dependency

### Phase 2

- automate proven bottlenecks
- retain analyst review

### Phase 3

- expand the pipeline only after retainers validate demand

## Guardrails

- no public overclaiming
- no automation-first messaging
- no video generation promise in V1
