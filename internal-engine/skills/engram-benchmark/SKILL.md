# ENGRAM Benchmark

This skill governs how the benchmark moat is built after enough audits exist.

## When to use it

- after 10 audits for internal pattern review
- after 25 audits for a first benchmark snapshot
- after 50 audits for a publishable report draft

## Core analysis questions

- which metrics show up most often
- what reveal timing patterns appear by category
- which hook types hold attention better
- what changes after clients implement fixes
- which score ranges correlate with stronger outcomes when data is shared

## Minimum schema assumptions

Track:

- category
- platform
- ad length
- input type
- metrics detected
- composite score
- zone ratings
- reveal timing
- hook type
- audio type
- implementation status
- performance outcome if available

## Quarterly report outputs

The report should cover:

- overall score distribution
- most common failure points
- hook analysis
- reveal timing analysis
- modality patterns
- trend shifts
- actionable recommendations
- methodology note

## Rule

Do not overstate the benchmark until enough rows exist. Internal review starts early; public authority is earned later.
