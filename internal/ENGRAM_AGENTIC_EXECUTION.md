# ENGRAM Agentic Execution
## Internal operating notes for recurring loops

This file explains how to run ENGRAM as a bounded agentic system.

## Principle

Every recurring loop must have:

- a trigger
- a source of truth
- a bounded output
- a review checkpoint
- a next action

## Approved loop families

### 1. Outreach intelligence

Input:

- pipeline
- pain signals
- competitor observations

Output:

- qualified target list
- battle card
- outreach email sequence
- next follow-up date

### 2. Trend and content

Input:

- trend observations
- category notes
- benchmark patterns

Output:

- short trend brief
- content hook set
- publishable draft

### 3. Audit to proof

Input:

- completed audit
- client reaction
- implementation notes

Output:

- proof summary
- case-study draft
- refreshed sales angle

### 4. Skill R&D

Input:

- active skill list
- repeated friction
- missing outputs

Output:

- skill refinements
- new skill candidates
- asset additions
- router updates

## Handoff standard

Use `assets/ops/agent_handoff_protocol.txt` for every workflow handoff.

## Guardrails

- no loop should silently change public positioning
- no loop should fabricate data
- no loop should remove human review from high-stakes outputs
- loops should create momentum, not hidden complexity
