# ENGRAM Tool Placeholders
## Reserved integration points for tools not yet live

This file exists so new tools can be added without rewriting the doctrine.

## Placeholder categories

- response modeling pipeline
- semantic scoring pipeline
- trend-ingestion helpers
- transcript and segmentation automation
- report population support
- rewrite generation support

## Rule for adding a new tool

A new tool should be documented only after answering:

1. What manual bottleneck does it remove?
2. What quality risk does it introduce?
3. Does the analyst still review the output?
4. Does it change the public offer? If not, keep it internal.

## Not-yet-live warning

Do not make future tools part of the required V1 workflow until they are actually in use and improving delivery.
