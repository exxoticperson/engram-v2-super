# PROJECT STATE
## ENGRAM status after the V2 Super architecture build

This file captures the current project state in a clean form so a fresh Codex session can understand where the project stands without relying on prior chat history.

## Current mission

ENGRAM is in build-stopped, execution-live mode.

The immediate goal is not more architecture.

The immediate goal is:

1. lock the compact report as the live proof asset
2. use it in outreach
3. run the first 50-agency batch
4. get replies and calls
5. close the first paid pilot

## Current business definition

ENGRAM is a manual pre-launch creative diagnostics service for short-form paid-social ads.

Current wedge:

- performance creative agencies
- Meta and TikTok paid social
- beauty and skincare DTC brands

Current public promise:

- where the ad loses the viewer
- why it happens
- what to change before spend

Not the promise:

- guaranteed results
- performance prediction
- software platform
- technical-model-led pitch

## What we decided during this build

- keep the wedge narrow
- keep the service manual first
- keep the stack internal in public messaging
- keep one pricing ladder
- keep one proof boundary
- keep one live metric system
- use the sample audit as the key proof asset
- avoid architecture theater
- prioritize execution and revenue over more internal expansion

## What is still unfinished

1. Real contact enrichment and outbound activity
   The first 50-agency batch now exists, but direct contacts, recent ad links, and reply history still need live population.

2. Booking path activation
   The booking spec exists, but the live link still has to be turned on outside the repo.

3. Manual payment path activation
   The repo now has the non-Stripe fallback path, but the actual Wise or bank details still need to be plugged in.

4. Live outreach results
   The system is prepared for execution, but the market has not answered yet.

## Current priority order

If a fresh operator or Codex session is deciding what to do next, use this order:

1. personalize and enrich the 50-agency batch
2. send the compact report through the first-client message pack

## Current risks

The main risks are:

- continuing to build instead of sending
- adding too many skills or loops without real usage data
- letting the proof asset stay "good enough" instead of flagship-quality
- drifting into broader markets too early
- design drift without one chosen language
